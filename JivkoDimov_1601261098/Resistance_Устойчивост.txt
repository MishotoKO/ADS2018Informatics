Persistent sorting methods are preferred when the elements of the set are already sorted according to some other criterion. For example, let us sort our file system by name and extension: files are named by name, like those with the same name ranks by extension. This can be done in two steps:
1) sort by extension
2) sort by name
While in the first sorting (by extension) algorithm stability is irrelevant in re-sorting (by name) the use of a sustainable method will already be mandatory.

Устойчивите методи на сортиране се предпочитат тогава когато елементите на множеството вече са сортирани съгласно някакъв друг критерий. Например нека искаме да сортираме файловата си система по име и по разширение: файловете се нареждат по име като тези с еднакво име се нареждат по разширение. Това може да стане на две стъпки: 
1) сортиране по разширение 
2) сортиране по име
Докато при първото сортиране (по разширение) устойчивостта на алгоритъма е без значение при повторното сортиране (по име) използването на устойчив метод вече ще бъде задължително.
